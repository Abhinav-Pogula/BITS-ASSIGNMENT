--- How Linux Manages File I/O ---
Every process in Linux is given a table of file descriptors, small 
integers that reference open files. By default, FD 0 is stdin, FD 1 is 
stdout, and FD 2 is stderr, all typically pointing to the controlling 
terminal. Redirection operators (>, >>, 2>, 2>&1) reassign these file 
descriptors to point at different destinations, such as regular files 
or /dev/null, without changing how a program itself writes -- the 
program still just writes to FD 1 or FD 2, and the shell decides where 
that data actually goes. Resource limits (ulimit) cap how many files a 
process can have open at once and other resource usage, protecting the 
system from runaway processes.
